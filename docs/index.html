简介

PocketBase 的 TypeScript 工具集，包含标准的 service/action/types 分层，支持用户、产品、公司资料等常用业务的接口与实现，适合二次开发和工程化扩展。

上传

# 1. 清空 dist（如有）
rm -rf dist

# 2. 重新构建
bun run build

# 3. 发布
npm publish --access public

# 或使用 np 自动处理版本、构建、发布
npx np

目录结构

src/
  base/                # 基础能力（集合、bootstrap等）
  utils/               # 工具类
  types/               # 类型定义（user.types.ts, product.types.ts, profile.types.ts）
  user.service.ts      # 用户相关接口定义
  user.action.ts       # 用户相关操作实现
  user.auth.ts         # 用户/管理员认证
  product.service.ts   # 产品相关接口定义
  product.action.ts    # 产品相关操作实现
  profile.service.ts   # 公司资料相关接口定义
  profile.action.ts    # 公司资料相关操作实现
  ...

快速使用

import { userService, productService, profileService } from "./services"
import { addUser, getAllUsers } from "./user.action"
import { addProduct, getAllProducts } from "./product.action"
import { addCompanyProfile } from "./profile.action"
import { setAdminAccount, loginAdmin, loginUser } from "./user.auth"

// 设置管理员账号
setAdminAccount({ email: "admin@xx.com", password: "yourpass" })
await loginAdmin()

// 用户操作
await addUser({ email: "test@xx.com", username: "test" })
const users = await getAllUsers()

// 产品操作
await addProduct({ title: "新产品" })
const products = await getAllProducts()

// 公司资料操作
await addCompanyProfile({
  companyName: "MyCo",
  contactEmail: "c@c.com",
  contactPhone: "123",
  companyAddress: "地址",
  allowRegistration: true,
})

类型体系

所有接口和数据结构均有独立类型定义，详见 src/types/ 目录。

Collection 实例导入规范

所有 pb 实例和 collection 实例（如 userCollect, productCollect, profileCollect 等）请统一从 src/base/index.ts 导入。

⚠️ 注意：src/base/collections.ts 仅为兼容旧代码保留，已废弃，后续将移除。请勿再从该文件导入。

示例：

import { pb, userCollect, productCollect, profileCollect } from "./base/index"
